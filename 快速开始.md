# ⚡ 5分钟快速部署指南

## 🎯 目标
将你的HTML演示文稿部署到GitHub Pages,获得一个可以分享的链接。

---

## 📝 步骤概览(3步完成)

### 第1步: 创建GitHub仓库 (2分钟)

1. **访问GitHub**
   - 打开浏览器,访问 [github.com](https://github.com)
   - 登录你的账号(没有就注册一个,免费的)

2. **创建新仓库**
   - 点击右上角的 "+" 号
   - 选择 "New repository"
   - 填写信息:
     ```
     Repository name: presentations
     Description: HTML演示文稿合集
     Public/Private: ✅ Public (必须公开)
     Initialize: ✅ Add a README file
     ```
   - 点击 "Create repository"

3. **复制仓库地址**
   - 创建后会看到一个绿色的 "Code" 按钮
   - 点击它,复制HTTPS地址
   - 类似: `https://github.com/你的用户名/presentations.git`
   - **保存这个地址**,马上要用

---

### 第2步: 上传文件 (2分钟)

#### 选项A: 使用命令行(推荐,更快)

打开终端(Mac: 在文件夹右键 → "新建位于文件夹的终端窗口")

```bash
# 进入项目目录
cd /Users/zhangxin/PyCharmMiscProject

# 初始化Git
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 关联GitHub仓库(替换成你的地址!)
git remote add origin https://github.com/你的用户名/presentations.git

# 推送
git branch -M main
git push -u origin main
```

如果提示登录:
- 会弹出浏览器
- 登录GitHub授权
- 自动完成推送

#### 选项B: 网页上传(更简单)

1. 在GitHub仓库页面
2. 点击 "Add file" → "Upload files"
3. 从Finder拖拽这些文件:
   - ✅ index.html
   - ✅ presentation*.html (所有5个)
   - ✅ notification_confirmation.html
   - ✅ README.md
   - ✅ 修改指南.md
4. 点击 "Commit changes"

---

### 第3步: 启用GitHub Pages (1分钟)

1. **进入设置**
   - 在仓库页面点击 "Settings" 标签

2. **找到Pages**
   - 左侧菜单向下滚动
   - 点击 "Pages"

3. **配置**
   - Source: Deploy from a branch
   - Branch: 
     - 第一个框选 `main`
     - 第二个框选 `/ (root)`
   - 点击 "Save"

4. **等待**
   - 大约1-2分钟
   - 刷新页面,会看到:
   - "Your site is live at https://..."

---

## 🎉 完成!

你的网站链接是:
```
https://你的用户名.github.io/presentations/
```

### 现在可以:
✅ 在浏览器中访问
✅ 在微信中分享链接
✅ 在手机上查看
✅ 分享给任何人

```bash
git add .

```

自动更新,无需重新配置!

## ❓ 遇到问题?

### 问题1: Git命令不会用?
👉 使用网页上传方式,更简单

### 问题2: 推送时认证失败?
👉 检查用户名和密码,或使用SSH密钥

### 问题3: Pages不显示?
👉 等待2-3分钟,清除浏览器缓存

### 问题4: 想要详细教程?
👉 查看 `GitHub部署教程.md`

---

## 💡 小贴士

1. **仓库必须公开**: GitHub Pages只对Public仓库免费
2. **必须有index.html**: 这是首页
3. **耐心等待**: Pages部署需要1-2分钟
4. **清除缓存**: 如果看不到更新,按Ctrl+Shift+R强制刷新
